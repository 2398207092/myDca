## 二、单元测试（最优先）

### 2.1 CostCalculatorTest

纯逻辑，不依赖 Spring，运行最快。

| 测试方法 | 测试场景 | 断言 |
|---------|---------|------|
| `calculateCostPerShare_diluted` | diluted 算法: 总买入10000, 总卖出0, 分红3000, 份额1000 | costPerShare = 7.0 |
| `calculateCostPerShare_dilutedOnly` | diluted_only 算法 | 不分红摊薄 |
| `calculateCostPerShare_weightedAvg` | 加权平均算法 | costPerShare = 买入均价 |
| `calculateDividendRate_normal` | 每份分红1元, 成本10元 | 10.00% |
| `calculateDividendRate_zeroCost` | 成本为0 | 0% |
| `calculateDividendRate_negativeCost` | 成本已收回(负成本) | -1 |
| `calculatePriceDividendRate_normal` | 每份分红1元, 市价20元 | 5.00% |
| `calculateRecoveryRate` | 累计分红3000, 净投入10000 | 30.00% |
| `calculateRecoveryYears` | 净投入10000, 已分红3000, 年分红2000 | 3.50年 |

## 四、数据对账 SQL（部署后手动跑）

```sql
-- 1. 市值对账：市值 ≠ 份额×最新价
SELECT name, shares, latest_price, market_value,
       ROUND(shares * latest_price, 2) AS expected_mv
FROM holdings
WHERE ROUND(shares * latest_price, 2) != market_value
  AND latest_price IS NOT NULL;

-- 2. 现金对账：预期流水 vs 实际余额（手动算，无初始现金时跳过）
-- 预期现金变动 = 总卖出到账 - 总买入支出 + 总已到账分红
```
