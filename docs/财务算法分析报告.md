## 一、核心算法总览

Fund Tracker 采用**三层计算架构**：

```
┌─────────────────────────────────────────┐
│  CostCalculator (纯计算引擎)            │
│  - 无状态，纯数学运算                   │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│  HoldingService.recalculateHoldingMetrics│
│  - 核心指标重算引擎                     │
│  - 聚合交易数据 → 调用 CostCalculator   │
└─────────────────────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    ▼               ▼               ▼
TransactionService  EventService    DataAuditor
(交易驱动重算)      (分红驱动)       (定时审计)
```

## 二、成本计算算法

### 2.1 三种成本算法

| 算法 | 公式 | 特点 | 适用场景 |
|------|------|------|----------|
| **diluted (分红摊薄)** | `(总买入 - 总卖出 - 累计分红) / 当前份额` | 将已收分红视为成本回收，成本最低 | 真实净投入视角 |
| **diluted_only (摊薄成本)** | `(总买入 - 总卖出) / 当前份额` | 不考虑分红，仅买卖净额 | 简单成本视角 |
| **weighted_avg (加权平均)** | `总买入金额 / 总买入份额` (永不小于0) | 只看买入端，卖出不影响 | 经典持仓成本 |

### 2.2 净投入计算

```
diluted:       净投入 = 总买入 - 总卖出 - 累计分红
diluted_only:  净投入 = 总买入 - 总卖出
weighted_avg:  净投入 = max(总买入 - 总卖出, 0)
```

**关键边界**: weighted_avg 的净投入永不小于 0，这导致当卖出金额 > 买入金额时，净投入显示为 0，成本息率无法计算。

## 三、收益率与息率算法

### 3.1 成本息率

```
成本息率 = 预测每股分红 / 每股成本 × 100%
```

**特殊处理**:
- 若每股成本 < 0（成本已完全收回）→ 返回 `-1`（前端展示为"已回本"）
- 若每股成本 = 0 → 返回 `0`

### 3.2 股价息率

```
股价息率 = 预测每股分红 / 最新价格 × 100%
```

### 3.3 回本进度

```
回本进度 = 累计已收分红 / 净投入 × 100%
```

## 四、预测分红算法

### 4.1 计算流程

```
1. 仅对 fund/ETF 类型持仓计算
2. 从数据库获取该基金所有分红记录
3. 取近3年记录（若3年内无记录则用全部）
4. 若无记录，尝试抓取天天基金分红数据
5. 调用 FundDividendScrapeService.calculateWithFrequency():
   a. 计算单次分红平均值 = 总分红 / 分红次数
   b. 识别分红频率（通过相邻除权日间隔）
   c. 年均每份分红 = 单次均值 × 年预期次数
6. 预测总分红 = 年均每份分红 × 持有份额
```

### 4.2 频率识别逻辑

通过相邻除权日间隔判断：
- 间隔 ≈ 1个月 → 月分红
- 间隔 ≈ 3个月 → 季分红
- 间隔 ≈ 6个月 → 半年分红
- 间隔 ≈ 12个月 → 年分红

## 五、回本年限算法

### 5.1 简单回本年限

```
回本年限 = max(净投入 - 累计已收分红, 0) / 预测年分红
```

- 若预测年分红为 0 → 返回 `999`（表示无穷大）

### 5.2 复投回本年限 ⭐ 核心创新算法

**模拟分红再投资的迭代过程**:

```
输入: 预测年分红, 当前份额, 已收分红, 净值

步骤:
1. 计算每年每份分红 = 预测年分红 / 当前份额
2. 初始化: cumulative = 已收分红, currentShares = 当前份额, years = 0
3. 循环 (最多100年):
   a. yearDividend = 每份分红 × currentShares
   b. cumulative += yearDividend
   c. 新增份额 = yearDividend / 净值
   d. currentShares += 新增份额
   e. years++
   f. 直到 cumulative >= 净投入
4. 返回 years
```

**假设条件**: 净值保持不变（保守估算）

**算法意义**: 考虑分红再投资带来的份额增长，比简单回本年限更准确。

## 六、交易处理算法

### 6.1 创建交易

```
1. 验证：卖出时检查份额充足
2. 计算总金额 = 数量 × 单价 + 手续费
3. 更新持仓份额:
   - buy/reinvest/bonus_share → shares += quantity
   - sell → shares -= quantity
4. 重新计算预测分红
5. 重新计算所有指标
6. 用最新价更新市值
7. 调整现金余额（买入扣减，卖出增加）
8. 异步刷新净值
```

### 6.2 份额从零重算

```
shares = Σ(买入/reinvest/bonus_share 份额) - Σ(卖出份额)
最终 shares = max(shares, 0)
```

## 七、现金记账算法

### 7.1 自动现金调整

```
1. 查找所有 type=cash 的资产
2. 选择主账户（isPrimary=true），无主账户则取第一个
3. target.amount += amount (正=增加，负=减少)
4. 由 @Transactional 调用方保障事务一致性
```

### 7.2 交易对现金的影响

| 交易类型 | 现金变动 | 说明 |
|----------|----------|------|
| 买入 | 减少 | 扣除购买金额 |
| 卖出 | 增加 | 增加卖出金额 |
| 删除买入 | 增加 | 反向恢复 |
| 删除卖出 | 减少 | 反向扣减 |
| 分红到账 | 增加 | 增加分红金额 |

## 八、数据审计算法

### 8.1 审计规则

| 优先级 | 规则 | 检查逻辑 | 级别 |
|--------|------|----------|------|
| P0 | 市值对账 | 市值 != 份额 × 最新价 | ERROR |
| P0 | 份额对账 | 持仓份额 != Σ买入 - Σ卖出 | ERROR |
| P1 | 现金流水对账 | 现金 != 卖出 - 买入 + 分红 | WARN |
| P1 | 分红异常 | 单次每份分红 > 近3年均值 × 3 | WARN |
| P1 | 息率异常 | 息率 > 30% 或 < 0 | WARN |
| P1 | 过期事件 | 发放日过7天仍为 pending | WARN |

### 8.2 现金流水对账公式

```
预期现金变动 = Σ卖出金额 - Σ买入金额 + Σ已到账分红金额
实际现金 = 当前所有现金资产金额之和
```

## 十、关键设计决策

1. **成本算法可切换**: 用户可在 diluted / diluted_only / weighted_avg 之间切换，适应不同投资视角
2. **预测分红保守**: 取近3年数据，避免异常历史数据干扰
3. **复投回本假设净值不变**: 保守估算，实际结果可能更优
4. **现金记账自动但非强制**: 自动调整现金，但失败不阻塞交易（事务回滚）
5. **审计定时运行**: 每天凌晨3点，不影响日间交易
