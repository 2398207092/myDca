# 对账审计功能方案

> 目的：自动发现数据不一致和异常，不需要事先知道 bug 在哪

---

## 设计原则

- **零侵入**：独立于业务代码，不改变任何现有逻辑
- **按严重级别分级**：ERROR（必须修）/ WARN（关注）/ INFO（记录）
- **可扩展**：每条规则独立，加新规则只需加一个方法

---

## 审计规则（按优先级）

### P0 — 数据一致性（ERROR 级别）

| 规则 | 检查逻辑 | 发现的 bug 类型 |
|------|---------|----------------|
| 市值对账 | `shares × latestPrice ≠ marketValue` 偏差 > 0.01 报错 | 交易后市值未刷新 |
| 现金流水对账 | 初始现金 + 卖出到账 - 买入支出 + 分红 = 当前现金 | 自动记账漏记/多记 |
| 交易份额对账 | 持仓份额 = 所有买入份额 - 所有卖出份额 | 交易记录不完整 |

### P1 — 数据合理性（WARN 级别）

| 规则 | 检查逻辑 | 发现的 bug 类型 |
|------|---------|----------------|
| 分红异常检测 | 单次分红 > 该基金均值 × 3 | 脏数据混入（你遇到的 0.25 分红） |
| 成本息率超限 | 成本息率 > 30% 或 < 0% | 成本计算错误 |
| 市值波动异常 | 单日市值变化 > 50% | 净值数据异常 |
| 预测分红突变 | 相比上次检查变化 > 50% | 分红重算逻辑错误 |

### P2 — 业务完整性（INFO 级别）

| 规则 | 检查逻辑 |
|------|---------|
| 净值缺失 | 持仓 x 天未更新净值 |
| 分红事件过期 | 分红发放日已过但状态仍为 pending |
| 定投计划过期 | 定投超过 x 天未执行 |

## 技术实现

### 方式一：定时任务（推荐，立即生效）

```java
@Component
@Slf4j
@RequiredArgsConstructor
public class DataAuditor {

    private final HoldingRepository holdingRepository;
    private final TransactionRepository transactionRepository;
    private final ManualAssetService manualAssetService;
    private final DividendEventRepository eventRepository;

    /**
     * 每天凌晨 3:00 执行全量对账
     */
    @Scheduled(cron = "0 0 3 * * ?")
    public void auditAll() {
        log.info("===== 开始数据对账 =====");
        
        List<String> errors = new ArrayList<>();
        List<String> warnings = new ArrayList<>();
        
        // P0: 市值对账
        auditMarketValue(errors);
        
        // P1: 分红异常检测
        auditDividendOutliers(warnings);
        
        // P1: 成本息率异常
        auditDividendRate(warnings);
        
        // ... 后续规则

        // 汇总报告
        if (!errors.isEmpty()) {
            log.error("对账发现 {} 个错误:", errors.size());
            errors.forEach(e -> log.error("  ❌ {}", e));
            // 可以在这里对接告警（钉钉/邮件等）
        }
        if (!warnings.isEmpty()) {
            log.warn("对账发现 {} 个警告:", warnings.size());
            warnings.forEach(w -> log.warn("  ⚠️ {}", w));
        }
        log.info("===== 数据对账完成，错误={}，警告={} =====", errors.size(), warnings.size());
    }
    
    private void auditMarketValue(List<String> errors) {
        holdingRepository.findByDeletedFalse().forEach(h -> {
            if (h.getLatestPrice() != null && h.getLatestPrice().compareTo(BigDecimal.ZERO) > 0) {
                BigDecimal expected = h.getShares()
                        .multiply(h.getLatestPrice())
                        .setScale(2, RoundingMode.HALF_UP);
                if (expected.compareTo(h.getMarketValue()) != 0) {
                    errors.add(String.format(
                        "市值不一致 [%s] shares=%s price=%s actualMV=%s expectedMV=%s",
                        h.getName(), h.getShares(), h.getLatestPrice(),
                        h.getMarketValue(), expected));
                }
            }
        });
    }
    
    private void auditDividendOutliers(List<String> warnings) {
        // 找出每只基金近3年分红中超过均值3倍的记录
        String sql = """
            SELECT r.fund_code, r.ex_date, r.dividend_per_share
            FROM fund_dividend_records r
            JOIN (
              SELECT fund_code, AVG(dividend_per_share) * 3 AS threshold
              FROM fund_dividend_records
              WHERE ex_date >= DATE_SUB(CURDATE(), INTERVAL 3 YEAR)
              GROUP BY fund_code
            ) a ON r.fund_code = a.fund_code
            WHERE r.dividend_per_share > a.threshold
              AND r.ex_date >= DATE_SUB(CURDATE(), INTERVAL 3 YEAR)
            """;
        // 执行 SQL 并记录警告
    }
}
```

### 方式二：API 触发（按需执行）

加一个 API 接口，随时手动触发：

```
GET /api/admin/audit
```

返回 JSON 格式的对账报告，可以在页面上展示。

## 上线后效果

以前：用户发现了 bug → 手动截图反馈 → 查代码定位 → 修

以后：

```
每天凌晨 3 点
  ↓
审计自动跑
  ↓
发现异常 → 日志 → 你看日志就知道哪里有问题
没发现异常 → 安心
```

你不需要写测试用例去猜哪里会出 bug，**只要审计规则覆盖了"正常数据长什么样"，不对的自然会报出来。**
